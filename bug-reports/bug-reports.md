| URL | https://www.saucedemo.com/inventory.html |
| Тест-кейс | TC-007 |

**Шаги:**
1. Открыть https://www.saucedemo.com
2. Авторизоваться: Username: problem_user, Password: secret_sauce

## BUG-002 [Medium] У problem_user поле Last Name на checkout не редактируется

| Поле | Значение |
|------|----------|
| Серьёзность | Medium |
| Приоритет | Medium |
| Модуль | Checkout |
| Пользователь | problem_user |
| URL | https://www.saucedemo.com/checkout-step-one.html |

**Шаги:**
1. Авторизоваться как problem_user
2. Добавить товар в корзину
3. Перейти в корзину → нажать «Checkout»
4. На странице checkout ввести First Name: John
5. Попытаться ввести Last Name: Doe

**Ожидалось:**
Текст «Doe» вводится в поле Last Name.

**Факт:**
Поле Last Name не принимает ввод. Курсор стоит в поле, но текст не появляется.

## BUG-004 [Major] У error_user нельзя удалить товар из корзины

| Поле | Значение |
|------|----------|
| Серьёзность | Major |
| Приоритет | High |
| Модуль | Корзина |
| Пользователь | error_user |

**Шаги:**
1. Авторизоваться как error_user
2. На странице каталога нажать «Add to cart» у любого товара
3. Перейти в корзину (нажать иконку корзины)
4. Нажать кнопку «Remove» у добавленного товара

**Ожидалось:**
Товар удаляется из корзины. Кнопка «Remove» меняется на «Add to cart».

**Факт:**
Ничего не происходит. Товар остаётся в корзине. В консоли ошибка:
`Uncaught TypeError: Cannot read properties of undefined`

## BUG-006 [Medium] Сортировка «Price (high to low)» не работает у error_user

| Поле | Значение |
|------|----------|
| Серьёзность | Medium |
| Приоритет | Medium |
| Модуль | Сортировка |
| Пользователь | error_user |

**Шаги:**
1. Авторизоваться как error_user
2. На странице каталога открыть выпадающий список сортировки (справа вверху)
3. Выбрать «Price (high to low)»

**Ожидалось:**
Товары отсортированы от самого дорогого к самому дешёвому:
$49.99 → $29.99 → $15.99 → $9.99 → $7.99

**Факт:**
Порядок товаров не меняется. Остаётся прежняя сортировка.
В консоли ошибка.

## BUG-007 [Minor] На странице checkout-complete.html нет кнопки возврата в каталог

| Поле | Значение |
|------|----------|
| Серьёзность | Minor |
| Приоритет | Low |
| Модуль | Checkout |
| Пользователь | standard_user |

**Шаги:**
1. Авторизоваться как standard_user
2. Добавить товар, перейти в корзину
3. Нажать Checkout, заполнить данные, нажать Continue
4. Нажать Finish
5. На странице «Thank you for your order!» посмотреть доступные действия

**Ожидалось:**
Понятная кнопка для возврата в магазин, например «Continue Shopping».

**Факт:**
Есть только кнопка «Back Home». Работает корректно, но название неочевидно
для интернет-магазина.
